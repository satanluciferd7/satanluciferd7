🇳🇵 Nepal Portal Implementation Guide

This page summarizes the main implemented features and the associated files.

1. Core Services

Key application structure and primary user-facing pages.

  • index.html - Main site landing page
  • news.html - Community news and updates
  • problems.html - Report and track public problems
  • services.html - Government service guides

2. Admin & RBAC

Admin tools, access control, and hidden utilities for secure setup.

  • admin.html - Admin dashboard and moderation
  • users.html - User management and role assignment
  • roles.js - Role-based access control logic
  • rbac-debug.html - Hidden debug tool for developers
  • reset-owners.html - Hidden owner reset utility
Note: The hidden pages rbac-debug.html and reset-owners.html are not linked publicly. Keep them out of the published navigation for production.

3. Feature Modules

JavaScript modules that power interactive features.

  • utils.js - Core utilities, data helpers, and storage wrappers
  • features.js - Feature-specific logic and UI helpers
  • service-wizard.js - Interactive government service wizard flows

4. Production Security

Security hardening steps for publishing.

  • Remove public links to hidden admin utilities
  • Separate RBAC logic from public code when ready
  • Use a backend authentication service in production
  • Keep roles.js as a protected layer and migrate the source of truth to a server-side database
